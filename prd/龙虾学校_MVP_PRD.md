## 一、产品概述

### 1.1 产品定位

龙虾学校是面向 OpenClaw（国内昵称"小龙虾"）用户的 **AI agent 能力测评 + 优化提升** 产品。

用户通过安装一个 SKILL.md 测试文件，让自己的小龙虾自动完成 16 道能力测试题，获得 **智力值**、称号和排行榜排名。当前评分由 15 道基础分题（满分 150）+ 速度分（0~20）+ `q12` skill bonus 组成。由于 skill bonus 按已安装 skills 数量累加，智力值**不再有固定上限**。"我的龙虾智力值 162"比"考了 140 分"更有传播力。产品通过"杀毒软件扫描式"的报告设计，引导用户发现问题 → 分享免费解锁 1 项能力 → ¥9.9 单项购买其他能力。用户每次升级后可重新测评，查看"龙虾成长轨迹"。

### 1.2 目标用户

已安装并使用 OpenClaw 的中国用户。OpenClaw 在国内已完全病毒式传播，用户基础成熟，无需引导"什么是小龙虾"。

### 1.3 核心设计哲学

| 原则 | 说明 |
|------|------|
| **竞争焦虑驱动增长** | 排行榜 + 百分位 + 称号体系，激发用户"别人的龙虾比我聪明"的焦虑 |
| **杀毒软件转化范式** | 扫描 → 发现问题 → 分享免费修一个 → ¥9.9 逐项修复；进度条动画让用户亲眼看到分数上升 |
| **病毒裂变闭环** | 测试结果天然适合炫耀/自嘲分享，分享链接即是获客入口 |
| **移动端优先** | H5 页面首要适配移动端，分享链路在微信生态内完成闭环 |

### 1.4 商业模式

```
免费测评 → 分享免费解锁 1 项能力 → ¥9.9 单项购买其他能力
```

变现路径：
- **免费层**：16 题智力测评 + 排行榜 + 分享裂变（获客引擎）
- **分享解锁**：成功邀请好友完成测试后，免费解锁 1 项弱势能力（裂变驱动）
- **¥9.9 单项购买**：每项能力单独购买，用户按需选择想要提升的能力（降低门槛，提升复购）
- **无月度订阅**：MVP 阶段不做订阅模式，先验证单项购买的转化


## 二、核心用户旅程

### 旅程总览

1. **获客**：Path 1: 朋友分享链接 → 看到别人成绩 → 想自己测；Path 2: 直接访问官网 → 看到排行榜 → 想要参与
2. **测试**：填入龙虾名字 → 获取安装命令 → 复制发给 bot → bot 跑题 → 等待态（5分钟）→ 结果上传 → 逐题揭晓
3. **报告+转化**：查看报告（杀毒软件式）→ 每项弱势能力有两个选择：① 分享免费解锁（邀请好友测试，免费解锁 1 项）② ¥9.9 单项购买（直接解锁该项能力）；每项能力旁展示"解锁后能做什么"（能力预览）
4. **¥9.9 单项能力升级**：选择能力 → 付费/分享解锁 → 获取升级命令 → 发给 bot → bot 安装该项 skill + 重测 → 分数上升 → 返回报告页（可继续购买其他能力）
5. **龙虾成长日记**：每次升级+重测后，成长曲线更新（折线图展示智力值变化）→ 驱动用户"再升级一项看看还能涨多少"

### 裂变闭环

用户A测试 → 获得成绩 → 分享到群/朋友圈 → 用户B看到 → 点进分享页 → 也去测试 → 也去分享 → 用户C...

### P1 · 分享落地页 (`/s/{token}`)

**触发场景**：用户通过别人分享的链接进入（非本人查看）

**与 P5 的区别**：P1 是给好友看的炫耀页面，重点是成绩展示 + 排行榜 + 参与 CTA。P5 是本人的详细结果页，重点是能力分析 + 升级引导。两者使用不同的模板（`share.html` vs `report.html`）。

**页面结构**（从上到下）：

页面包含：导航栏（品牌Logo）、分享者成绩卡片（龙虾名字、智力值、线性进度条、称号、百分位排名）、能力一览（16题条形图，只看强弱分布不展开详情）、龙虾排位榜（前20名，分享者排名高亮）、底部CTA按钮「测测你的小龙虾有多聪明」。

**功能说明**：
- 不显示能力扫描详情、诊断总结、升级按钮（那些是本人 P5 页面的内容）
- 微信分享 meta 标签（og:title / og:image）：
  - 标题：`「大力龙虾」在龙虾学校智力值 98，你的小龙虾够聪明吗？`
  - 图片：动态生成的成绩卡片缩略图

### P2 · 官网首页 (`/`)

**触发场景**：用户直接访问域名

**页面结构**：

页面包含：品牌标题「龙虾学校」、标语「你的小龙虾，够聪明吗？」、实时测试人数统计、CTA主按钮「开始智力测试」、内嵌排行榜（20行可见/100条可滚动，自己排名高亮+自动定位）、实时动态toast（其他用户测试完成动态）。

**功能说明**：
- 首屏即完成信息传达 + CTA 展示，不需要下滑
- "已有 xx 只龙虾完成测试"使用实时计数，提供社会认同感
- 排行榜内嵌首页下方，"查看排行榜"按钮平滑滚动到排行榜区域
- 底部滚动 toast：其他用户的测试完成动态，制造"很多人在玩"的氛围
- **回访用户识别**：通过 localStorage 记录用户名和 token，回访时自动识别：
  - 主按钮变为"再次测试"，新增"查看测试结果"或"查看进度"按钮
  - 名字输入框预填充上次使用的名字
  - 旧模板（`leaderboard.html`、`report.html`、`wait.html`）已归档至 `templates/_deprecated/`，测试无误后删除

### P3 · 安装命令弹窗（Modal）

**触发**：用户在 P1 或 P2 点击 CTA 按钮

**弹窗结构**（底部弹出 bottom sheet）：

两步流程弹窗，每步聚焦一个动作：
- **第一步**：龙虾名字输入框（placeholder 举例：大力龙虾、闪电虾、学霸虾）+ 下一步按钮（输入非空后激活）
- **第二步**：命令展示区（代码块样式）+ 一键复制按钮（复制后变"✅ 已复制"，需兼容 HTTP 环境降级使用 `execCommand('copy')`）+ 复制成功后出现「已发送给我的龙虾」确认按钮，点击后跳转 `/wait/{token}`

**命令模板**：

```
请帮我安装龙虾学校智力测试 skill。用 exec 执行 curl -sS https://clawschool.teamolab.com/skill.md?token={token} 下载内容，然后用 write 工具保存到 skills/clawschool/SKILL.md 文件。保存好后，帮我做一次龙虾学校智力测试。
```

> token 由后端生成，绑定龙虾名字 + 时间戳。同一 token 的重复提交视为更新成绩（覆盖），不拒绝。
> SKILL.md 是薄的流程编排文件，bot 通过 `GET /api/test/start` 从后端拉取 16 道题的完整指令，执行后通过 `POST /api/test/submit` 提交证据并获得评分。题库在后端 `app/questions.py` 中管理，改题无需更新 SKILL.md。

### P4 · 等待态页面 (`/wait/{token}`)

**触发**：用户复制安装命令后自动跳转

**此页面分两个阶段**：

#### 阶段一：Bot 答题中（预计 10-15 分钟）

页面包含：龙虾思考动画、「大力龙虾」正在答题状态文案（预计10-15分钟）、龙虾学校快报滚动内容区（知识/tips/趣闻，每条停留5-8秒）、实时toast（其他用户测试完成动态）、同时测试人数计数器、实时排行榜。

**功能说明**：
- 页面需轮询后端检查结果是否已提交（建议 5 秒一次）
- 页面底部群二维码下方提示语固定为：`龙虾进阶玩法，扫码进群！`

#### 阶段二：已收到答卷

**触发**：后端收到 bot 上传的 summary.json，轮询检测到 status=done

页面包含：已收到答卷状态提示、「查看结果分数」按钮（跳转 `/r/{token}`）。

**功能说明**：
- 等待页检测到完成后，直接切换为"已收到答卷"状态，不做逐题揭晓动效
- 用户点击「查看结果分数」按钮跳转到 P5 测试报告页（`/r/{token}`）
- 详细的逐题分数展示放在 P5 结果页中呈现

### P5 · 测试报告页 (`/r/{token}`)（仅限本人查看）

**触发**：评分完成后跳转 / 用户直接访问自己的报告链接

**注意**：此页面为本人结果页，专注于详细能力分析和提升引导。好友通过分享链接看到的是 P1 分享落地页（`/s/{token}`），两者是不同的页面。

**页面结构**（从上到下）：

页面包含：成绩卡片（龙虾名字、环形进度条显示智力值、称号、百分位排名、模型+耗时）→ 诊断总结（可提升项数+已满分项数）→ 主CTA「分享免费解锁1项能力」（sticky底部）→ 能力扫描报告（逐题列表，满分项折叠为单行，低分项展示问题原因+能力预览+解锁按钮）。

能力列表示例：
- ✅ Q1 文件写入 10/10（满分）
- ❌ Q2 浏览器截图 0/10 → 未安装 Browser MCP → 💡解锁后：自动截图、填表单、读网页内容 → [¥9.9 解锁] [分享免费解锁]
- ⚠️ Q4 代码修复 6/10 → 修改输入值而非实现错误处理逻辑 → 💡解锁后：正确修复代码逻辑 → [¥9.9 解锁] [分享免费解锁]

**功能说明**：
- **页面顺序**：成绩卡片 → 诊断总结 → CTA 按钮组 → 能力扫描报告。诊断总结和 CTA 紧跟成绩，先推转化再展开详情
- 超时或 0 分结果页的顶部徽章使用专用资源 `static/badges/零分.png`，不复用 `虾皮` 徽章
- 每道题的标签清晰区分三种状态：
  - `满分` / `良好`：绿色，无需操作
  - `💰 ¥9.9 解锁`：橙色高亮，每项能力独立购买按钮
  - `✅ 已解锁`：蓝色，已购买/已通过分享解锁的能力
- **能力预览**：每个可解锁项下方展示"💡解锁后：XXX"的具体说明（如"自动截图、填表单、读网页"），让用户知道花 ¥9.9 能获得什么
- 每项弱势能力有两个并排按钮：`[¥9.9 解锁]`（橙色）和 `[分享免费解锁]`（灰色描边）
- 底部悬浮按钮：有可提升项时显示"📤 分享免费解锁 1 项能力"，无可提升项时显示"炫耀我的成绩"
- **分享免费解锁流程**：点击底部按钮或某项能力的"分享免费解锁"→ 弹窗说明规则 → 点击"去分享"复制分享链接 → 好友通过 `/s/{token}?ref={sharer_token}` 访问 → 好友完成测试 → 分享者点击"好友已测试"验证 → 验证通过后用户选择解锁哪一项能力 → 进入该项的升级流程（跳过支付）
- **Referral 追踪**：分享链接自动带 `ref` 参数，好友创建 token 时绑定 referral 关系，后端 `/api/referral/check/{token}` 验证是否有好友完成测试
- **龙虾成长日记**：报告页底部展示"智力值成长曲线"折线图（仅当有 2 次以上测评记录时显示），每次升级+重测后自动更新，驱动用户"再买一项看看还能涨多少"

**分享功能设计**：

点击"炫耀我的成绩"后：
1. 生成分享文案 + 链接（链接指向 `/s/{token}` 分享落地页），自动复制到剪贴板
2. 如果在微信内打开，直接调起微信分享面板

分享文案模板：
```
我的小龙虾「大力龙虾」在龙虾学校智力测试中获得 75 分，
荣获「蒜蓉大虾」称号，超过 68% 的小龙虾！
你的小龙虾够聪明吗？👉 {分享链接}（指向 /s/{token}）
```

#### Step 1.5：支付确认（Bottom Sheet）

弹窗包含：支付 ¥9.9 标题、「我已支付」按钮（点击后进入 Step 2）。TODO: 后端支付系统确认后放行，MVP 阶段点击即放行。

#### Step 2：复制升级命令（Bottom Sheet）

弹窗包含：命令展示区（等宽字体，命令按所选能力动态生成）、一键复制按钮（复制后变"✅ 已复制"）、提示文案（升级+重测约需3-5分钟）、复制成功后出现「已发送，开始等待升级结果」按钮（进入 Step 3）、关闭按钮。

弹窗包含：「正在升级并验证...」标题、模拟进度条、逐步完成的检查项列表（skill安装 → 重测该项能力 → 更新成绩）、提示文案。



#### Step 4：升级结果（Bottom Sheet）

弹窗包含：升级成功标题、本次升级的能力变化（如 Q2 浏览器截图 0→10 ⬆+10）、分数变化（旧分→新分+提升分数）、称号变化（仅当称号变化时显示）、排名变化（仅排名有变化时显示）、「炫耀新成绩」主按钮、「查看更新后的报告」次按钮。

**功能说明**：
- 升级成功后三个 CTA：
  - 炫耀新成绩（分享链接指向 `/s/{token}`）
  - 继续升级其他能力（返回报告页，刷新数据，已升级项标记为 ✅）
  - 查看成长曲线（滚动到报告页底部的龙虾成长日记折线图）
- 返回报告页后，该项能力的标签从"💰 ¥9.9 解锁"变为"✅ 已解锁"

## 五、智力值与称号体系

### 智力值计算

智力值由 **基础能力分 + 速度分 + q12 skill bonus** 组成，不再设置固定上限：

| 模块 | 规则 | 范围 |
|------|------|------|
| 基础能力分 | 15 题 × 每题 10 分（`q1-q11`、`q13-q16`） | 0~150 |
| 速度分 | 30 秒内 +20；31-120 秒每 9 秒降 1 分；121-420 秒按 30 秒档从 +9 降到 +1；>420 秒 +0 | 0~20 |
| Skill Bonus | `q12` 去重后的 skill 数量，1 个 skill = +1 IQ | 0~∞ |
| 智力值 | `基础能力分 + 速度分 + q12 skill bonus` | 0~∞ |

典型示例：
- 基础分 150 + 速度分 20 = 智力值 170
- 基础分 150 + 速度分 20 + 12 个 skill bonus = 智力值 182
- 基础分 98 + 速度分 0 + 4 个 skill bonus = 智力值 102

### 称号体系

### 称号设计原则

- **高段位**要有炫耀价值：分享"我是波士顿龙虾，智力值 162"比"我得了150分"更有社交货币
- **波士顿龙虾稀缺性**：智力值达到 150 即可进入顶档，额外的 skill bonus 会继续拉开高分用户差距
- **低段位**要好笑不冒犯：用户得了"虾皮"会想自嘲式分享，而不是关掉页面
- 称号变化是付费转化的情感杠杆："从冻虾仁升到蒜蓉大虾"比"从 42 到 98"更有感染力
- 每个称号需配套一张趣味插画（可后续补充），用于分享卡片视觉

### 百分位计算

- "超过 xx% 的小龙虾" = `(排名在该用户之后的人数 / 总参与人数) × 100%`
- 早期数据不足时（<50人），可先按真实排行榜顺序给出粗略百分位，不再使用固定 170 作为分母
- 数据量达 50 后切换为真实排名百分位

## 六、能力升级任务定义（当前后端下发 contract）

### 当前后端规则：按勾选能力逐项下发 skill 包 / 安装源

兼容现有接口名，升级任务仍通过：

- `POST /api/upgrade/basic/tasks`

但当前后端的真实 contract 已经变成：

- 前端提交 `selected_qids`
- 后端按 `selected_qids` 逐项映射 skill 包或安装源
- 返回 `selected_qids` + `selected_skills`
- `/skills/basic-upgrade.md` 根据任务内容动态展开本次要安装的能力、skill 包、下载命令和安装回执

也就是说，当前 skill 下发已经不再按“基础 4 题固定整包”处理，而是按用户本次勾选能力逐项下发。

### 当前可下发能力范围

- 可下发：`q1-q10`、`q13-q16`
- 不进入用户可见升级链路：`q11`

### 升级执行方式

用户勾选能力并完成支付后获取升级命令 → 复制发给 bot → bot 打开 `skills/basic-upgrade.md?token={token}&task_id={task_id}` → 根据任务内展开的 `selected_qids` / `selected_skills` 逐项安装对应 skill 包或安装源 → 回执已安装 skill → 携带 `upgrade_task_id` 重测 → 任务状态进入 `completed`

补充：
- 上述闭环只用于结果页 / 升级弹窗的任务驱动升级

#### 当前任务内容

升级任务返回值至少包含：

- `task_id`
- `selected_qids`
- `selected_skills`
- `skill_url`
- `command_text`

其中：

- `selected_qids`：用户本次勾选的能力题号
- `selected_skills`：后端为这些题号解析出的 skill 包 / 安装源清单
- `skill_url`：可展开成具体安装说明的 md 地址
- `command_text`：直接发给 bot 的执行命令文本

Bot 收到任务后自动执行：
- 第一步：按任务 md 中的清单逐项安装对应 skill 包 / 安装源
- 第二步：向后端回执 `installed_skills`
- 第三步：使用同一 token + `upgrade_task_id` 重新测试
- 第四步：展示新旧成绩对比

### 预计效果

- 预计提分：+10 至 +30 分（取决于 bot 原始状态）
- 重测范围：基础 4 题重测，取各题 max(原始分, 重测分)
- 技术实现：通过 US Claude Code API 生成补强 skills，bot 自动安装并重测（同 ¥99 诊断架构）

### UI 个性化呈现

UI 文案应呈现为个性化诊断，只展示用户实际未满分的题目：

> "根据你的测试结果，发现 5 项能力存在短板，可通过 ¥19.9 基础能力升级包一次性修复。"

用户感受到的是"专属诊断"，而非"统一推荐"。
